## 1. Oracle cloud의 무료 범위

우리가 대부분 사용하는 클라우드 서비스는 free tier를 제공하고 있다. AWS, GCP, Azure 등 대표적인 여러 클라우드 서비스들은 프리티어를 제공하여 사용자들이 간단하게 테스트 할 수 있게 도와주고 테스트가 끝나면 비용을 지불해서 추가로 사용할지 결정하면 된다.

하지만, 많은 1인 개발자나 포트폴리오를 만들고자 하는 사용자들은 AWS 프리티어를 많이 사용하고, 종종 GCP 나 Azure같은 서비스에서 프리티어를 쏙 빼먹는 일종의 보트피플을 감행하기도 한다. 그러는 와중에서 유명한 클라우드가 Oracle cloud가 한 3년전 쯤 부터 급부상하기 시작했다. 생각보다 좋은 프리티어를 제공하기 때문.

제공하는 `주요 무료 범위`는 다음과 같다.

* **인프라**
  * AMD 기반 컴퓨트 VM 2개(각각 OCPU 1/8개\*\* 및 1GB 메모리)
  * Arm 기반 Ampere A1 코어 및 24GB 메모리(매월 OCPU 3,000시간 및 18,000GB-시간을 제공하며 VM 1개 또는 4개로 사용 가능)
  * 블록 볼륨 스토리지 2개, 총 200GB

\
많은 사용자들이 ARM 컴퓨트 인스턴스를 최대 4코어에 24GB 를 제공해준다는 점 때문에 오라클 클라우드를 선택하곤 한다. 요즘에는 국내 리전에서의 free tier의 제공 인스턴스 용량이 다 소진되어서 Out of capacity가 발생하는 문제가 있는데, 이건 PAYG를 성공하면 문제없이 사용 가능하다.

그 외에도 이번에 작성하는 OKE에 대해서는 PAYG로 계정 전환한 유저는 `Oracle Cloud Infrastructure Kubernetes Engine - 기본 클러스터` 가 무료로 제공되기 때문에 간단한 서버나 포트폴리오, 1인 개발자 서버로는 매우 충분하고 무료로 사용할 수 있다.\
(굳이 돈을 내지 않고도 K8S를 구성할 수 있고, 굳이굳이 ARM 서버의 2CPU를 컨트롤 플레인으로 낭비하지 않아도 되며, AWS EKS 처럼 관리형 K8S를 사용할 수 있다는 매우 좋은 장점이 있다.)

그 외에도 OCI에는 유료계정 전환시에 사용할 수 있는 추가적인 무료 범위들이 있으니 필요한 것은 찾아보면 좋을 듯 하다.

## 2. OKE 클러스터 구성 스펙

우선 무료 범위에서 쿠버네티스를 구성할 예정이기 때문에 스펙은 다음과 같이 정했다.

* Control Plane : Oracle Cloud Infrastructure Kubernetes Engine - 기본 클러스터 / 무료
* Worker Node : Ampere A1, 2CPU 12GB RAM \* 2 / 4 CPU, 24GB RAM 까지는 무료 범위
  * Boot volume : 50GB / 최소 50GB 부터 설정 가능하고 200GB 까지는 무료이며 웬만해선 인스턴스 용량을 초과하지 않는다.
  * 혹시라도 DB 서버나 간단한 AMD 서버를 열어야 할 때 AMD에서도 사용해야 할 부트 볼륨이 필요하니 아끼는게 좋고, 쓸 일이 절대 없다면 100GB 두개 또는 1CPU 6GB RAM 으로 4개의 노드를 구성해서 각각 50GB 씩 주는 방법도 가능하다.

## 3. OKE 클러스터 구성하기

![](/images/oci-k8s-cluster-1/image.png)

일단 콘솔에서 oke를 검색하면 서비스 - Kubernetes 클러스터(OKE) 메뉴가 나오니 클릭해서 들어간다.

![](/images/oci-k8s-cluster-1/image.png)

클러스터 생성 버튼을 누르면 빠른 생성과 사용자정의 생성이 나오는데, 네트워크에 관련된 리소스 옵션을 하나하나 조절해서 만들어야 하는게 아니라면 빠른 생성을 누르고 하단의 진행을 누른다.

![](/images/oci-k8s-cluster-1/image.png)

이름은 원하는 클러스터 이름을 선택하고 다음과 같이 설정한다.

* 이름 : 사용하고자 하는 이름으로 마음대로 설정
* 컴파트먼트 : 사용하고자 하는 compartment 설정, 따로 없다면 루트로 해도 됨
* Kubernetes 버전 : 안정적인 버전에서 가장 최신버전으로 설정, 26-03-04 기준 1.34.2가 가장 최신의 버전이다.
* Kubernetes API 끝점(endpoint) : public endpoint를 선택한다. 무료 범위에선 배스천 서버나 연결을 위한 리소스를 더 쓰기엔 아쉽다
* 노드 유형 : 무조건 관리됨 선택해야한다. 가상으로 선택하면 향상된 K8S인 enhanced로 설정될 가능성이 높고 유료 요금이 부과될 수 있다.
* 워커 노드 : 프라이빗 노드로 선택한다. 보안적인 부분에서 프라이빗 서브넷을 통해서 노드에 접속하는게 안전하다.
* 구성 및 이미지 : 우측 `구성 변경`을 눌러서 노드 구성에 VM.Standard.A1.Flex 를 선택한다.&#x20;
* OCPU : 2, 메모리 : 12GB로 구성 (위 OKE 클러스터 스펙 내용 참조)
* 이미지는 구성 변경시 가장 최신 버전으로 자동 매칭됨
* 노드 수 : 2 (만약 1CPU 6 Ram 으로 4개 노드 구성시에는 4로 입력해도 됨)
* 고급 옵션
  * 부트 볼륨 ON : 50GB (만약 AMD 서버나 4개 노드를 사용할 계획이 없다면 100GB로 설정해도 됨)
  * 전송 암호화, 볼륨 암호화는 선택. 굳이 암호화 기능이 필요없다면 OFF
* 이미지 확인 OFF
* SSH 키 추가 : SSH 키 쌍 생성 (개인적으로 만들고 싶다면 직접 만들어도 됨.)
  * 프라이빗, 퍼블릭 키는 무조건 다운로드 받아서 보관해두어야 한다.
* kubernetes 레이블 : 필요한 태그 있다면 추가. 나는 따로 하지 않는다

이 정도 설정이 되었다면 다음을 누르고 클러스터 생성을 진행한다. 생각보다 클러스터 생성이 간단하게 진행되고 생성되는데는 노드 세팅이나 네트워크 세팅 등 여러가지 이유로 인해서 시간이 조금 걸린다. 생성이 완료되면 그 후에 추가적인 세팅이 가능하다.

## 4. 로컬에 OCI cli 설치 및 접속 세팅

AWS 를 사용해도 로컬에서 이것 저것 건드리려면 AWS cli를 설치해서 진행해야한다. Oracle cloud도 마찬가지, 매번 cloud shell에 접속해서 해결할 수 없으니 OCI cli를 설치하도록 한다.

![](/images/oci-k8s-cluster-1/image.png)

OKE 클러스터 리스트에서 내가 생성한 OKE 클러스터에 들어가면 상단에 `빠른 시작: 샘플 앱 배치` 가 있다. 메뉴에 진입하면 내 클러스터에 엑세스라는 메뉴 오른쪽에 `클러스터에 엑세스`를 클릭 시 메뉴가 나오며 `로컬 엑세스` 선택 시 설정 방법을 알 수 있다. 아래 다운로드 및 설치를 클릭하여 로컬에서 OCI cli를 설치하도록 하자

현재 이걸 구성하는 내 PC는 맥 OS 환경이니 home brew를 통해서 설치하도록 한다.

```bash
brew update && brew install oci-cli
```

설치가 완료되면 해당 메뉴에 나와있는 명령어들을 그대로 따라하게 되면 oci cli 사용 세팅이 완료된다. 엑세스 구성은 퍼블릭 엔드포인트를 선택했으니 당연히 퍼블릭 명령어를 쳐서 사용한다.\
\
![](/images/oci-k8s-cluster-1/image.png)

정상적으로 잘 연결되었는지 확인하기 위해서는 위에 명령어를 따라하면 되는데, 애플리케이션 배치는 정상동작 하지 않는다. deployment.yaml을 보면&#x20;

```yaml
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
```

image의 nginx 부분이 명시적으로 적혀있지 않기 때문이다.&#x20;



```bash
cat <<EOF | kubectl apply -f -
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: docker.io/library/nginx:latest
        ports:
        - containerPort: 80
EOF
```

\
이런식으로 command를 사용하면 정상적으로 동작한다. oracle cloud에서도 저건 좀 수정된 버전을 이용하던지 했음 좋겠다.
