## 1. kubernetes에 argocd 설치



GitOps를 구현하기 위해서는 그에 맞는 여러 툴이 필요한데 쿠버네티스 환경에서는 argocd가 가장 유명한 툴임에는 분명하다.&#x20;

[https://argo-cd.readthedocs.io/en/stable/getting\_started/](https://argo-cd.readthedocs.io/en/stable/getting_started/)

```bash
kubectl create namespace argocd
kubectl apply -n argocd --server-side --force-conflicts -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

argocd 사이트를 참고해서 설치하면, argocd라는 namespace와 pod가 생성된다.

```
kubectl get pods -n argocd
```

위 명령어를 입력하면 생성된 argocd pod들을 볼 수 있다. 정상적으로 Running이 된다면 정상으로 설치가 된 것. 대시보드에서 세팅이 필요하다.

```
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
```

를 입력하면 대시보드 로그인을 위한 비밀번호가 나온다. 임의의 문자열을 복사해두자. 현재는 argocd를 접속하기 위한 세팅이 아무것도 되어있지 않기 때문에 포트 포워딩을 통한 argocd 대시보드에 접근한다.

```
kubectl port-forward svc/argocd-server -n argocd 8080:443
```

포트 포워딩 명령어를 통해 127.0.0.1:8080으로 접속하면 대시보드 화면을 볼 수 있다. 아이디는 admin, 비밀번호는 아까 복사해 둔 문자열을 입력한다.

![](/images/oci-k8s-cluster-2/1.png)

![](/images/oci-k8s-cluster-2/2.png)

로그인이 완료되면 왼쪽 User Info 에 들어가서 UPDATE PASSWORD를 통해 관리자 비밀번호를 변경하도록 한다. 굳이 cli를 이용하지 않아도 변경이 가능하다. \
`kubectl -n argocd delete secret argocd-initial-admin-secret`

보안을 위해서 위 명령어를 통해서 secret 키를 삭제하면 기본적인 argocd 설치와 세팅이 완료된다.



## 2. ArgoCD 및 여러 앱 접속을 위한 traefik 설치

이전에는 NGINX Ingress를 이용해서 접속을 하도록 세팅을 하였으나, NGINX Ingress는 26년 3월부로 지원을 종료한다고한다. 그래서 대안으로 찾은것이 traefik을 통한 라우팅을 선택했다. traefik IngressRoute를 통해서 특정 도메인 접속 라우팅이 지원되도록 설정을 하고자한다.

서비스를 구성할 때 Oracle cloud를 이용하였지만 가능하면 무료로 쿠버네티스를 구성하기 위한 상태이기 때문에 로드 밸런서를 이용할 때 무조건 Network LoadBalancer 만 이용하도록 한다. Oracle cloud는 Load Balancer는 유료이지만, Network LoadBalancer(NLB)는 무료로 제공하고있다. 나는 굳이 helm을 로컬에 설치하지 않고 argocd를 이용해서 설치하는 방식을 채택하여 설치하고자 한다.



![](/images/oci-k8s-cluster-2/3.png)

ArgoCD 메인페이지의 대시보드 왼쪽 상단에 NEW APP 버튼을 누른뒤 다음과 같이 입력한다.

```
Application Name : traefik
Project Name : default (클릭하면 하단에 노출)
SYNC POLICY : Automatic
ENABLE AUTO-SYNC 
PRUNE RESOURCES
SELF HEAL 세 개 모두 체크

AUTO-CREATE NAMESPACE 체크
```



![](/images/oci-k8s-cluster-2/4.png)



```
Repository URL : https://traefik.github.io/charts/   / 우측 HELM선택
Chart : traefik  / 우측 버전 최신으로 선택
```



![](/images/oci-k8s-cluster-2/5.png)

```
Cluster URL : https://kubernetes.default.svc (클릭하면 자동 선택)
Namespcae : traefik
```



![](/images/oci-k8s-cluster-2/6.png)

```
service:
  enabled: true
  type: LoadBalancer
  annotations:
    oci.oraclecloud.com/load-balancer-type: "nlb"
```

Oracle cloud에서 무료로 제공하는 로드밸런서인 NLB를 사용하도록 선택한다. 이 후 상단의 create를 누르면 싱크가 돌고 자동으로 oci에 NLB를 생성해준다. `kubectl get pods -n traefik` 명령어로 traefik 네임스페이스에 정상적으로 파드가 돌고있는지 확인할 수 있다.
